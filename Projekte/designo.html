Projekt: Designo

Eine Webseite für eine fiktive Designagentur mit insgesamt sieben Seiten, basierend auf einer Challenge von Frontend-Mentor.

Beschreibung

Dies ist meine Lösung zu einer weiteren Frontend-Mentor-Challenge: "Designo designo-multipage-website". Mithilfe eines JavaScript-Frameworks bzw. einer Library soll das Figma-Design umgesetzt werden. Hierzu habe ich die React-Library gewählt. Die Webseite umfasst ein Hamburger-Menu in der Mobile-Version, eine Startseite, drei Seiten zu den Themen "Web Design", "App Design" und "Graphic Design" Darüber hinaus gibt es noch eine Seite mit Informationen über die Agentur, eine Seite mit einer Übersicht von den verschiedenen Standorten und eine Seite mit Kontaktformular.

Die folgenden Anforderungen sollten am Ende erfüllt sein:

  • Optimales Layout für die jeweilige Bildschirmgröße
  • Hoverstates für alle interaktiven Elemente
  • Formularvalidierung

Tech Stack

HTML5, CSS3, JavaScript (ES6), React.js, React Hooks, React Router, Styled Components

Schwierigkeiten und Lösungen

  1. Formularvalidierung: Die Inputfelder im Kontaktformular sollten ihre Text- und Randfarben ändern, sobald ein User anfängt, es auszufüllen. Außerdem sollte noch ein Icon mit einem Ausrufezeichen und die Wörter "Can't be empty" angezeigt werden, wenn ein Inputfeld beim Abschicken nicht ausgefüllt ist. Zunächst habe ich die Werte jedes Inputs in einem Zustand gespeichert. Der Anfangszustand war jeweils ein leerer String. Über ein onChange-Event bei jedem Inputfeld konnte ich dann diesen Zustand mit den neu eingetippten Werten aktualisieren. Die Error-Meldung habe ich ebenfalls über Zustände organisiert, die jeweils prüfen, ob ein Inputfeld leer oder nicht ist. Dieser Vorgang wird durch eine handleSubmit-Funktion beim Auslösen des onSubmit-Events durchgeführt.
  2. Mobile-Version von der Navigation: In der Mobile-Version sollte die Navigation versteckt und nur über einen Hamburger-Button erreichbar sein. Wenn man auf diesen klickt, kommt das Navigationsmenu von der linken Seite ins Bild gefahren. Hierzu habe ich den Checkbox-Trick verwendet. Das Inputfeld vom Typ "checkbox" wird mit einem Label verknüpft. Dieses Label beinhaltet dann das Icon, in diesem Fall sogar zwei Icons: Hamburger-Icon und X-Icon. Ersteres wird angezeigt, wenn das Menü noch nicht sichtbar ist. Letzteres wenn das Menü sichtbar ist und wieder geschlossen werden kann. Dazu habe ich einen Ternary-Operator benutzt, der das entsprechende Icon bzw. die Komponente anzeigt. Die Änderung des Icons findet über einen Zustand statt, in dem gespeichert wird, ob die Checkbox geklickt oder ungeklickt ist.
